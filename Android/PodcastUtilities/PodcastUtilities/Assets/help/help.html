PodcastUtilities Cache Manager

PodcastUtilities is an RSS feed media cache management application. You can

• Download new episodes from a group of feeds
• Remove old episodes from the cache
• Generate playlists from episodes in the cache
• Use it in conjunction with the desktop version

Note that it can only perform cache management - PodcastUtilities cannot play media you will need a separate player.

Configuring the downloads

The PodcastUtilities engine PodcastUtilities.Common.DLL is configured and controlled by an XML control file. There are two mechanisms for changing the configuration

• Edit the XML control file directly, usually on a destop computer
• Modify the configuration in this app

You can use both mechanisms interchangeably

Editing the XML control file directly

You can tweak the control file to your liking using the desktop version in conjunction with the documentation..

When you move the XML control file across to your phone (possibly email it to yourself) there are some elements you should take care of

sourceRoot should be set to the root of your cache on the phone, probably /storage/emulated/0/Podcasts

playlistPathSeparator should probably be/

Do remember that when you load the control file into this app then the app takes a copy of the control file, if you make any further edits to the file you will need to load it again.

Generally the usage pattern is to share your existing control file, via email or the file system, edit the XML and then load the control file back in.

Configuring within the application

The configuration screens are a work-in-progress, also they do not give access to every item in the control file.

The Cache Root

For a phone its probably easiest to use /storage/emulated/0/Podcasts.

If you are using Windows Subsystem for Android and want to use the shared Windows File System then you will need to change your SourceRoot to be something like /storage/emulated/0/Windows/Podcasts

Getting Started

To add new podcast feeds to the configuration follow these steps

• Configure - Select Configure from the main screen
• Cache Root - Set the cache root to suite you, ensure the global values and feed defaults are set how you want them
• Add a new Podcast - Select + to add a new podcast feed, give it a name, the name must be unique
• Add feed RSS URL - Select the new feed and enter a valid RSS feed
• Configure Podcast - Override any of the feed defaults for this specific feed

Podcast utilities uses RSS feeds to access podcast episodes. Podcast sites provide links to podcasts and often these links only work with their own app, you need to look for something called RSS, sometimes the following icon is used

Downloading episodes

Generally the usage pattern is

• Purge Podcasts - to get rid of old media
• Download Podcasts - to get new media
• Generate Playlist - to get a playlist of the downloaded media

You could elect to delete the media files manually and skip the purge, also if you use a folder player for your media then you may not need a playlist.

Keyboard

If a keyboard is present, then the following keyboard shorcuts are available on the main screen

• P - Purge Podcasts
• D - Download from all Feeds
• G - Generate Playlist
• S - Settings
• C - Configure

You can also use the ESCAPE key as the back button.

Using with the desktop version

If you use the desktop version (DownloadPodcasts and SyncPodcasts) to manage the cache on your phone and you want to use this app in addition to that mechanism then it should work fine, there are some considerations

• Marke sure you use the same naming style on both platforms or the filenames will not match and you might end up with duplicates
• Try to ensure you use the same version of PodcastUtilities.Common.DLL on both platforms or the filenames might differ because of differences in the code.

Not on Google Play?

To install PodcastUtilities you can get the APK from the _PreBuiltPackages folder on Github. We did attempt to distribute it via the Google Play Store however as we needed "Manage Storage" permission Google declined saying "The feature you identified that is dependent on this permission does not appear to be critical to the core functionality of your app", rather than arguing the toss with a Google-bot we decided to not list the app there.

You can install any of my apps using Obtainium without an app store.
